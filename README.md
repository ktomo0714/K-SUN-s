## 🎯 概要

**おみせ開業AI**は、飲食店の開業を検討している方向けの無料シミュレーションツールです。

ユーザー登録不要で、簡単な情報入力だけで以下の情報を即座に取得できます：

- 💡 **コンセプト提案**: あなたの店の特徴や差別化ポイントを明確化
- 📊 **KPI予測**: 売上・来客数・回転率などの重要指標を算出
- 💰 **収支予測**: 初期投資から月次収支、損益分岐点まで試算
- 🎯 **戦略提案**: マーケティング、オペレーション、メニュー戦略

### 🌟 特徴

- **登録不要**: メールアドレスや個人情報の入力は一切不要
- **3ステップで完了**: ジャンル選択 → 基本情報入力 → 結果表示
- **40以上の業種対応**: 和食、洋食、カフェ、アジア料理など幅広くカバー
- **立地別シミュレーション**: 駅前、オフィス街、住宅街など5つの立地タイプに対応
- **リアルな数値**: 飲食業界の実データに基づく信頼性の高いシミュレーション

## 🚀 主な機能

### 1. ジャンル選択
飲食店のメインカテゴリとサブカテゴリを選択します。

**対応カテゴリ（全6カテゴリ）:**
- 🍣 **和食**: 寿司、天ぷら、ラーメン、うどん・そば、居酒屋、焼鳥、とんかつ、定食屋
- 🍝 **洋食**: イタリアン、フレンチ、ステーキ、ハンバーグ、洋食屋
- ☕ **カフェ**: コーヒー専門店、スイーツカフェ、ランチカフェ、ナイトカフェ
- 🥟 **アジア料理**: 中華料理、韓国料理、タイ料理、ベトナム料理、インド料理
- 🍔 **ファストフード**: ハンバーガー、ピザ、チキン、弁当・惣菜
- 🍸 **バー・居酒屋**: バー、ワインバー、クラフトビール、立ち飲み

### 2. 基本情報入力
開業予定の店舗に関する基本情報を入力します。

**入力項目:**
- 席数（1〜500席）
- 客単価（100〜100,000円）
- 営業時間（例: 11:00-22:00）
- 立地イメージ（駅前、オフィス街、住宅街、商業施設内、ロードサイド）
- ターゲット顧客（任意）
- 特徴・こだわり（任意）

### 3. シミュレーション結果表示

#### 💡 コンセプト提案
- 店舗名案の提示
- 立地と特徴を活かしたコンセプト文
- ターゲット顧客へのメッセージ
- 差別化ポイント（3〜4項目）

#### 📊 KPI予測
- 月間売上予測
- 日次来客数
- 回転率
- 平均客単価
- 原価率、人件費率、家賃比率

#### 💰 収支予測
- 初期投資額
- 月間固定費・変動費
- 損益分岐点（投資回収期間）
- 年間利益予測

#### 🎯 戦略提案
- **集客戦略**: 立地に応じたマーケティング施策
- **オペレーション戦略**: 効率的な運営のための施策
- **メニュー提案**: メニュー構成や価格戦略

## 🚀 環境構築・起動方法

### 前提条件

以下のソフトウェアがインストールされている必要があります：

- **Node.js**: 20.0以上
- **npm**: 10.0以上（Node.jsに同梱）
- **Git**: 最新版

### 1. リポジトリのクローン

```bash
# HTTPSでクローン
git clone https://github.com/ktomo0714/K-SUN-s.git

# または SSHでクローン
git clone git@github.com:ktomo0714/K-SUN-s.git

# ディレクトリに移動
cd K-SUN-s
```

### 2. 依存パッケージのインストール

```bash
# omise-aiディレクトリに移動
cd omise-ai

# 依存パッケージをインストール
npm install
```

### 3. 開発サーバーの起動

```bash
# 開発サーバーを起動
npm run dev
```

起動が成功すると、以下のようなメッセージが表示されます：

```
  ▲ Next.js 14.2.0
  - Local:        http://localhost:3000
  - Environments: .env.local

 ✓ Ready in 2.3s
```

### 4. ブラウザでアクセス

ブラウザで以下のURLにアクセスしてください：

```
http://localhost:3000
```

### 5. その他のコマンド

```bash
# 本番ビルド
npm run build

# 本番サーバー起動（ビルド後）
npm start

# リンターの実行
npm run lint
```

## 🔧 シミュレーションロジックの概要

### 業種別パラメータ

各業種ごとに以下のパラメータを設定しています：

- **原価率**: 売上に対する食材原価の割合（22%〜38%）
- **人件費率**: 売上に対する人件費の割合（24%〜30%）
- **家賃比率**: 売上に対する家賃の割合（8%〜12%）
- **回転率**: 1日あたりの座席回転数（1.5〜5.0回）
- **初期投資/席**: 1席あたりの初期投資額（35,000〜100,000円）

### 立地補正

立地によって以下の補正を適用：

| 立地 | 家賃補正 | 来客数補正 |
|-----|---------|-----------|
| 駅前・駅近 | 1.3倍 | 1.2倍 |
| オフィス街 | 1.2倍 | 1.1倍 |
| 住宅街 | 0.8倍 | 0.9倍 |
| 商業施設内 | 1.5倍 | 1.3倍 |
| ロードサイド | 0.7倍 | 1.0倍 |

### 計算例

**ラーメン店（20席、客単価1,000円、駅前）の場合:**

```
日次来客数 = 20席 × 4.0回転 × 1.2（立地補正） = 96人/日
月間売上 = 96人 × 1,000円 × 25日 = 2,400,000円
月間利益 = 売上 - (原価 + 人件費 + 家賃 + その他) = 513,600円
初期投資 = 20席 × 50,000円 = 1,000,000円
投資回収期間 = 1,000,000円 ÷ 513,600円 ≒ 2ヶ月
```

詳細は「バックエンドシミュレーションロジック説明書.md」を参照してください。
